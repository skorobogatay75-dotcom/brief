# Brief & КП — Платформа для брифов и коммерческих предложений

Платформа для IT-разработчиков: создавайте проекты, отправляйте клиентам ссылку на бриф, после заполнения автоматически формируется красивое коммерческое предложение (лендинг).

## Возможности

- **Админ-панель** — создание проектов, данные клиента, учёт всех заявок в таблице
- **Редактируемый бриф** — настраиваемые поля формы с переменными для КП
- **Уникальные ссылки** — автоматическая генерация ссылок на бриф и КП
- **Авто-КП** — после заполнения брифа формируется лендинг с ответами клиента
- **Статусы** — черновик → бриф отправлен → заполнен → КП готово
- **152-ФЗ** — шифрование ПДн, исключение из ИИ-генерации, согласие клиента

## Требования

- [Node.js](https://nodejs.org/) 18.18 или новее
- npm

## Установка

```bash
# 1. Установите зависимости
npm install

# 2. Создайте файл окружения
copy .env.example .env

# 3. Инициализируйте базу данных
npm run db:push

# 4. Запустите сервер разработки
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000)

**Пароль по умолчанию:** `admin123` (измените в `.env` → `ADMIN_PASSWORD`)

**Ключ шифрования:** задайте `ENCRYPTION_KEY` (минимум 32 символа) для защиты персональных данных по 152-ФЗ.

## Защита персональных данных (152-ФЗ)

| Механизм | Реализация |
|----------|-----------|
| Шифрование | AES-256-GCM для ФИО, email, телефона и полей брифа с меткой ПДн |
| Разделение данных | Публичные ответы и ПДн хранятся отдельно (`briefAnswers` / `encryptedBriefPii`) |
| Генерация без ПДн | Этап 1: шаблон заполняется плейсхолдерами `[[PII:key]]` |
| Подстановка после | Этап 2: ПДн подставляются локально на сервере, без ИИ |
| ИИ-интеграция | Используйте `prepareAiGenerationInput()` — ПДн исключены гарантированно |
| Согласие | Обязательный чекбокс на форме брифа, фиксация `pdConsentAt` |

Подробнее: [/privacy](/privacy)

## Как пользоваться

### 1. Создайте проект
`/admin` → «Новый проект» → укажите название, описание и данные клиента.

### 2. Настройте бриф
В карточке проекта → вкладка «Бриф» → редактируйте поля формы.  
Каждое поле имеет **ключ переменной** (например `project_goal`), который подставляется в шаблон КП как `{{project_goal}}`.

### 3. Отправьте ссылку клиенту
Скопируйте ссылку на бриф из таблицы или карточки проекта.  
Формат: `http://localhost:3000/brief/ТОКЕН`

### 4. Клиент заполняет бриф
После отправки клиент автоматически перенаправляется на коммерческое предложение.

### 5. Просмотрите результат
- В таблице проектов — статус, ссылки, даты
- В карточке проекта → вкладка «Ответы» — все ответы клиента
- Ссылка на КП: `http://localhost:3000/proposal/ТОКЕН`

## Переменные для шаблона КП

| Переменная | Описание |
|---|---|
| `{{project_title}}` | Название проекта |
| `{{project_description}}` | Описание проекта |
| `{{client_name}}` | Имя клиента |
| `{{client_email}}` | Email клиента |
| `{{client_phone}}` | Телефон |
| `{{client_company}}` | Компания |
| `{{project_goal}}` | Цель проекта (из брифа) |
| `{{target_audience}}` | Целевая аудитория |
| `{{main_features}}` | Основные функции |
| `{{timeline}}` | Сроки |
| `{{budget}}` | Бюджет |
| `{{design_refs}}` | Референсы |
| `{{comments}}` | Комментарии |

## Структура проекта

```
src/
  app/
    admin/          — панель администратора
    brief/[token]/  — форма брифа для клиента
    proposal/[token]/ — лендинг КП
    api/            — REST API
  components/       — UI-компоненты
  lib/              — утилиты, БД, авторизация
  types/            — TypeScript типы
prisma/
  schema.prisma     — схема SQLite
```

## Деплой

1. Установите `NEXT_PUBLIC_APP_URL` на ваш домен
2. Смените `ADMIN_PASSWORD`
